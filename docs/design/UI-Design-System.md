## 1. 设计哲学

### 1.1 核心原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **数据驱动** | 一切以数据展示为核心，减少装饰性元素 | 指标卡 > 装饰图 |
| **高效操作** | 最短路径完成任务，减少点击次数 | 快捷入口、批量操作 |
| **一致体验** | 所有模块遵循统一规范 | 筛选区、表格、弹窗统一 |
| **渐进披露** | 复杂信息分层展示 | 折叠面板、详情抽屉 |
| **容错设计** | 操作可撤销，错误有提示 | 二次确认、撤销提示 |

### 1.2 设计目标

- **效率**：90% 操作在 3 步内完成
- **清晰**：数据一目了然，趋势快速识别
- **专业**：企业级 SaaS 质感，建立信任感
- **可扩展**：支持未来模块无缝接入

- 默认显示"名称 + 关键标识"
- 联动时上游变化 → 下游清空 + 重新拉取

## 7. 状态设计

### 7.1 加载状态

| 场景 | 表现 |
|------|------|
| **页面首次加载** | 全屏 Skeleton（模拟表格 6 行 / 卡片 3 行） |
| **局部加载** | 组件级 Skeleton 或 Loading 图标 |
| **表格加载** | 6 行 Skeleton + 列宽匹配 |
| **卡片加载** | 3 行 Skeleton |
| **树形加载** | 3 层 Skeleton 节点 |

### 7.2 空状态

| 场景 | 表现 |
|------|------|
| **列表为空** | 居中插图 + "暂无数据" + 引导操作按钮（如适用） |
| **筛选无结果** | 插图 + "该筛选下暂无数据" + "重置筛选"按钮 |
| **待办为空** | 插图 + "暂无待办" |
| **快捷入口为空** | 不显示该卡片 |

**空状态插图风格**：扁平化插画，主色调 `#C0C4CC`，尺寸 160×120px

### 7.3 错误状态

| 场景 | 表现 |
|------|------|
| **页面级错误** | 顶部 Banner（红色）+ "加载失败，请稍后重试" + 重试按钮 |
| **组件级错误** | 组件内显示错误提示 + 重试 |
| **网络超时** | Toast "网络超时，请检查网络连接" |
| **无权限** | 整页 "无访问权限，请联系管理员" |

### 7.4 无权限状态

| 场景 | 表现 |
|------|------|
| **按钮无权限** | 置灰 + tooltip "无权限" |
| **页面无权限** | 整页 "无访问权限，请联系管理员" |
| **数据无权限** | 按数据权限范围过滤（BR-006 四级） |
