## 1. IP 组状态机

### 1.1 状态定义

| 状态值 | 名称 | 含义 |
|--------|------|------|
| 0 | 停用 (DISABLED) | 不可见、不可操作，但数据保留 |
| 1 | 启用 (ENABLED) | 正常可用 |

### 1.2 状态转移

```
创建 ─▶ 启用(1)
       │
       │ [管理员操作]
       ▼
       停用(0)
       │
       │ [管理员操作]
       ▼
       启用(1)
```

### 1.3 转移约束

- 启用 → 停用：无限制（停用后子项不可见，但数据保留）
- 停用 → 启用：检查其下成员/账号/主播的有效性（无效则提示）
- 删除：仅允许 停用状态 + 无子数据 → 删除（具体业务校验见 API 1005）

### 1.4 副作用

| 转移 | 副作用 |
|------|--------|
| 启用→停用 | 该 IP 组下账号对**数据采集**模块**隐藏**；数据**保留** |
| 停用→启用 | 重新对**数据采集**模块**可见** |

## 3. 数据补录状态机（关键 ⭐）

### 3.1 状态定义

| 状态值 | 名称 | 含义 |
|--------|------|------|
| 0 | 待审核 (PENDING) | 补录已提交，等待审核 |
| 1 | 审核通过 (APPROVED) | 已合并入 `oa_content_daily`，**不可改/不可删** |
| 2 | 审核不通过 (REJECTED) | 被驳回，补录人可修改后重新提交 |

### 3.2 状态转移图

```
                  ┌────────────────────────────────────┐
                  │                                    │
[补录人提交]       │                                    │
       │          │                                    │
       ▼          │                                    │
  ┌─────────┐    │      [审核人]          ┌──────────┐ │
  │ 待审核  │ ──▶│ ─────────────────────▶ │ 审核通过 │ │ (终态)
  │  (0)    │    │      reviewStatus=1    │   (1)    │─┘
  │         │    │                        └──────────┘
  │         │    │
  │         │    │      [审核人]          ┌──────────┐
  │         │ ──▶│ ─────────────────────▶ │ 已驳回   │──┐
  │         │    │      reviewStatus=2    │   (2)    │  │
  └─────────┘    │                        └──────────┘  │
       │         │                              │        │
       │         │                              │        │
       │         │                              │ [补录人]│
       │         │                              │ 修改+  │
       │         │                              │ 重新提交│
       │         │                              ▼        │
       │         │                         ┌─────────┐   │
       │         │                         │ 待审核  │ ◀─┘
       │         │                         │  (0)    │
       │         │                         └─────────┘
       │         │
       │         │ [补录人]   待审核可撤回
       │         │ ───────────▶ 已撤回(3)
       │         │                │
       │         │                ▼
       │         │            (记录保留，仅不入统计)
       │
       └──▶ [补录人撤回] ──▶ 已撤回(3)
```

### 3.3 转移矩阵

| From | To | 触发者 | 条件 | 副作用 |
|------|----|--------|------|--------|
| (新) | 待审核(0) | 补录人 | 提交补录请求 | 写入 `oa_content_data_import` (reviewStatus=0) |
| 待审核(0) | 审核通过(1) | 审核人 | 点击"审核通过" | 合并入 `oa_content_daily` (data_source='IMPORT') |
| 待审核(0) | 已驳回(2) | 审核人 | 点击"审核不通过" + 备注 | 更新 `review_status=2`, `review_remark` |
| 待审核(0) | 已撤回(3) | 补录人 | 点击"撤回" | 软删除（`deleted_at`） |
| 已驳回(2) | 待审核(0) | 补录人 | 修改后重新提交 | 创建新 record (reviewStatus=0)，关联旧 record ID |

### 3.4 不可转移规则

| From | To | 拒绝原因 |
|------|----|----------|
| 审核通过(1) | 任何 | 已通过记录不可改、不可删、不可再审 |
| 已驳回(2) | 审核通过(1) | 必须先重新提交（创建新 record），不可直接审 |
| 已撤回(3) | 任何 | 终态（仅作记录） |

### 3.5 数据合并规则

审核通过时，调用 `ContentDailyMergeService.merge(contentId, statDate)`：

1. **去重**：若 `oa_content_daily` 已有同 `(content_id, stat_date)` 记录
   - 保留 API 采集数据
   - 补录数据**仅覆盖** `data_source='IMPORT'` 字段
2. **统计合并**：聚合时 `data_source IN ('API', 'IMPORT')` 的记录都计入
3. **追溯**：补录记录 `id` 写入 `oa_content_daily.import_record_id` 字段，便于反向追溯

### 3.6 字段级权限

| 字段 | 待审核(0) | 审核通过(1) | 已驳回(2) | 已撤回(3) |
|------|-----------|-------------|-----------|-----------|
| `read_count/like_count/...` | 补录人可改 | ❌ 不可改 | 补录人可改 | ❌ 不可改 |
| `review_status` | 审核人可改 | ❌ 不可改 | 审核人可改 | ❌ 不可改 |
| `review_remark` | - | 审核人写 | 审核人写 | - |

## 4. 运营→主播关联生命周期

### 4.1 状态机

无显式状态（启用/停用），但有时间区间约束。

### 4.2 关键规则

- `start_date` ≤ `end_date`（或 `end_date` 为 null 表示"长期有效"）
- **重叠检查**：同一对 `(ops_user_id, anchor_user_id)` 的多个关联**不可有时间重叠**
  - 即对于任意两条 (A1, A2)，`A1.start_date < A2.end_date AND A1.end_date > A2.start_date` → 拒绝
- 人效盘点中按"时间区间 + 该期间关联的 IP 组"聚合

## 5. 状态机实现要点

### 5.1 后端实现

- 使用 **Spring State Machine** 或自研 `StatusMachine` 工具类
- 所有状态转移通过 `transition(fromStatus, toStatus, operator, context)` 统一入口
- 转移前必须校验：权限 + 业务规则 + 副作用触发

### 5.2 前端展示

- 状态值使用 i18n key（中文/英文）
- 状态转移按钮可见性：根据"当前状态 × 角色 × 业务规则"动态计算
- 状态机操作日志：写入 `oa_audit_log` (action='STATE_TRANSITION', from=X, to=Y)

### 5.3 测试覆盖

- 单元测试：覆盖每条转移路径
- 异常路径：转移条件不满足时返回明确错误码
- 回归：每次状态机变更需跑全量测试

### 与 M4 账号管理的关键关联

M1 中所有平台账号必须通过 M4 的"实名人/手机/手机卡/公司"选择：

```
M1 IP 组 → M1 账号 → M4 实名人（强关联）
                     → M4 手机（强关联）
                     → M4 手机卡（强关联）
                     → M4 公司（强关联）
```

**校验**：
- 已停用实名人/手机/手机卡 不可被新账号引用（错误码 1501）
- 已绑定到其他账号的实名人/手机/手机卡 需"强制替换"（错误码 1502）
- 跨租户过滤（错误码 1504）

### 错误码

| 错误码 | 含义 |
|--------|------|
| 1500 | 关联的实体不存在 |
| 1501 | 关联的实体已停用/注销 |
| 1502 | 关联的实体已被其他记录引用 |
| 1503 | 字典值不合法 |
| 1504 | 跨租户访问禁止 |

详见 [`GLOBAL-CONVENTIONS.md § 5.3`](../engineering/GLOBAL-CONVENTIONS.md)
