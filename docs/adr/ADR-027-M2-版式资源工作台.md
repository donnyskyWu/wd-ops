# ADR-027：M2 版式资源工作台（96 等效 Route C）

| 字段 | 值 |
|------|---|
| 编号 | ADR-027 |
| 标题 | M2 内容编辑器 — 自建版式资源工作台（样式库 + 模板库 + 一键排版） |
| 状态 | **Accepted** |
| 日期 | 2026-06-16 |
| 关联 | ADR-020 · S-15~S-17 · Route C |

---

## 1. 决策

在 M2 文章编辑器右侧集成 **版式资源工作台**，三 Tab：

| Tab | 数据源 | 能力 |
|-----|--------|------|
| 样式 | `oa_layout_style` | 分类浏览、预览、插入光标 / 应用到选中 |
| 模板 | `oa_wechat_layout_template` | 全部使用 / 分开使用 / 使用背景 + 参数滑块 |
| 一键排版 | `oa_typesetting_rule` | 规则勾选、对比预览、应用 |

工作台可折叠，偏好存 `localStorage`。

## 2. 后端

- **V89** `oa_layout_style` + 30 条 seed + `oa:layout-style:*` 权限
- **V90** 模板表扩展 `tags`、`default_params`
- **V91** `oa_typesetting_rule` + 5 条 seed + `oa:content:typeset`
- `LayoutMergeService` 扩展：`paramOverrides`、`includeBlockTypes`（partial-apply）、`backgroundOnly`
- 排版服务 **铁律**：`plainTextBefore === plainTextAfter`（ADR-020）

## 3. 前端

- `LayoutResourceSidebar` 集成于 `ContentEditPanel`（ARTICLE + 非只读）
- `RichTextEditor` 暴露 `insertHtmlAtCursor` / `applyStyleToSelection`

## 4. 增强（2026-06-16）

### 4.1 一键排版双模式

| 模式 | API `mode` | 行为 |
|------|-----------|------|
| 按模板排版 | `TEMPLATE` | `templateId` + `paramOverrides` → `LayoutMergeService.merge` |
| 智能优化排版 | `AUTO` | `SMART_OPTIMIZE` 规则 + 样式库 styleRefs + 子规则序列 |

`ContentTypesetReq` 扩展：`body`、`templateId`、`mode`、`paramOverrides`。  
规则 `ruleConfig.linkedTemplateId` 支持 `TEMPLATE_LINK` 类型（可选 seed）。

### 4.2 模板提取高保真（2026-06-16 修订）

**导入策略（视觉 SSOT + 套用分离）**

| 阶段 | 存储/输出 | 用途 |
|------|-----------|------|
| 导入 | `preview_html` / `layout_html` = 消毒后**完整**文章 HTML（含 inline style、装饰区块） | 向导预览、列表缩略、模板库空正文预览 — **对齐 96 编辑器「模板即全文视觉效果」** |
| 导入 | `layout_schema` + `style_css` | 套用时的 merge 骨架；**不**将样本正文写入内容 |
| 套用 | `LayoutMergeService.merge(body, schema)` | 正文保真（ADR-020）；输出 `layout_json` / `layout_html` |

实现：`LayoutImportExtractor` — `rawHtml` 全量保留 → `previewHtml`；并行 `extractLayoutSchemaFromLayoutJson` 供 merge。

**仍保留的骨架能力**

- HTML 解析 inline style → `globalStyles` enrichment
- 装饰块识别 → `fixed` 槽位
- Word 导入识别 Heading1/2/3
- `POST /layout-template/validate-extract-fidelity` 样本 merge 验证
- 修复 `preview-merge` 的 `paramOverrides` 与 `defaultParams` 合并

**局限**：套用输出仍由 schema + globalStyles 渲染，复杂嵌套/section 装饰在 apply 阶段可能无法 100% 还原；预览与套用视觉可能存在差距，后续可通过 styleRef→导入 HTML 片段克隆增强。
