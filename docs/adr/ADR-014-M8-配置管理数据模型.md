# ADR-014: M8 配置管理数据模型映射

> **状态**: 已采纳 | **日期**: 2026-06-11
> **决策人**: 开发团队
> **关联**: `配置管理模块PRD.md` v1.0

## 背景

新版 `配置管理模块PRD.md` 定义 11 张 `config_*` 表及 `/api/config/*` 路径。现有实现使用 4 张 `oa_*` 表 + `/admin-api/oa/config/*`，并已通过 GATE-S2 验收。

## 决策

**保留 `oa_*` 四表架构**，通过列扩展 + 2 张补充表映射 PRD 概念，不重建 schema。

| PRD 表 | 实现表 | 映射方式 |
|--------|--------|----------|
| `config_internal_account` | `oa_collect_config` (scope=INTERNAL) | `config_name`→账号名称；`account_identifier`→账号标识；`app_id`/`app_secret_encrypted`；快手专用字段 |
| `config_aocreate_api` | `oa_aocreate_api` | 独立表，管理员专用 |
| `config_external_account` | `oa_collect_config` (scope=EXTERNAL, sub_type=account) | 外部账号监控 |
| `config_keyword` | `oa_config_keyword` | 独立表 |
| `config_order_db` | `oa_collect_config` (scope=GENERAL) | `db_*` 字段组 |
| 外部数据源 | `oa_collect_config` (scope=EXTERNAL_SOURCE) | 保持现有 |
| `config_threshold_alert/fans/work/override` | `oa_threshold_config` | `threshold_category` 区分四类 |
| `config_ai_model` | `oa_ai_model_config` | 扩展 `model_id`/`timeout`/`is_default`/`conn_status` |
| `config_ai_prompt` | `oa_ai_prompt_config` | 扩展 `version` |

## API 路径

PRD 示例路径 `/api/config/internal/accounts` **不采用**；统一保持 Yudao 前缀 `/admin-api/oa/config/{subdomain}/*`，在 API-M8 中标注 PRD→实现映射。

## 铁律遵守

- 全表 `tenant_id` 隔离
- 凭证字段 AES-256 + 响应脱敏
- 枚举 `@InDict` + 错误码 1503
- 跨租户 1504

## 后果

- 正面：无需数据迁移、IT 用例可渐进扩展
- 负面：`oa_collect_config` 列较多，需 scope 约束字段使用

## 补充（2026-06-11 v2.1）

### 账号统一

- `oa_collect_config.account_id` FK → `oa_account.id`（平台类内部采集 Tab）
- 前端 `<AccountSelect />` 选账号；`CollectConfigServiceImpl` 列表 join `accountName`
- 种子：`V50__seed_m8_collect_accounts.sql` 关联 account 9001–9010；`V51` 清理 V43 无 `account_id` 遗留

### 企微 / 个微 Tab 拆分

| Tab | 数据 | API |
|-----|------|-----|
| 企业微信 | `oa_wework_account` | `/oa/internal/wework/*`（复用 `WeworkAppConfigPanel`） |
| 个人微信 | `oa_aocreate_api` | `/config/internal-collect/aocreate` |

### 外部采集 sub_type

- 外部账号：`scope=EXTERNAL`, `sub_type=account`, `platform_type` 用 `dict_third_platform`
- 关键词：独立表 `oa_config_keyword`
