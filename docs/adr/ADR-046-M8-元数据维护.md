# ADR-046：M8 配置管理 — 元数据维护

| 字段 | 值 |
|------|---|
| 编号 | ADR-046 |
| 标题 | 元数据实体与字段查询条件配置 |
| 状态 | **Accepted** |
| 日期 | 2026-06-18 |
| 关联 | M6 指标 / 自定义查询 · M8 配置管理 |

---

## 1. 背景

指标构建器与自定义查询当前依赖前端硬编码 `metricSchema.ts`。需将物理表映射为**元数据实体**，并为每个字段配置**指标查询条件类别**，供查询组件自动渲染（IP 组、人员、平台、账号、日期、文本、枚举等）。

## 2. 决策

### 2.1 数据模型

| 表 | 说明 |
|----|------|
| `sys_metadata_entity` | 实体：`entity_code`、`entity_name`、`physical_table`、`status` |
| `sys_metadata_field` | 字段：`field_code`、`column_name`、`data_type`、`query_condition_type`、`dict_type`、`selector_config`、`sort` |

- 全表 `tenant_id` 隔离（铁律 #3）
- 枚举 `@InDict`：`dict_metadata_query_condition_type`、`dict_metadata_entity_status`

### 2.2 API

前缀：`/admin-api/oa/metadata`

| 方法 | 路径 | 权限 |
|------|------|------|
| GET | `/list` | `oa:metadata:query` |
| GET | `/unmapped-tables` | `oa:metadata:query` |
| GET | `/table-columns` | `oa:metadata:query` |
| POST | `/create` | `oa:metadata:create` |
| GET | `/{id}` | `oa:metadata:query` |
| PUT | `/update` | `oa:metadata:update` |
| DELETE | `/{id}` | **`ROLE_OA_ADMIN`**（超级管理员） |
| PUT | `/{entityId}/fields` | `oa:metadata:update` |
| GET | `/entity/{code}/fields` | `oa:metadata:query`（指标/自定义查询集成读口） |

### 2.3 未映射表发现

`information_schema.TABLES` 减去已映射 `physical_table`；排除 `sys_*`、`flyway_*`。

### 2.4 超级管理员

项目 seed 超级管理员角色 code = **`OA_ADMIN`**（authority `ROLE_OA_ADMIN`）。删除元数据实体仅此角色可执行。

## 3. 后果

- 正面：M6 可逐步替换 `metricSchema.ts` 为 API 驱动
- 负面：首版未做指标侧自动消费，仅暴露读口
