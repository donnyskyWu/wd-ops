# ADR-002: 前端规范源选择

> **状态**：✅ Accepted
> **日期**：2026-06-07
> **决策者**：产品/技术
> **关联**：[`TECH-CONSTRAINTS.md § 1.2`](../engineering/TECH-CONSTRAINTS.md) | [`开发规范/UI设计与开发规范.md`](../../../开发规范/UI设计与开发规范.md)

---

## 1. 背景

项目存在两份前端规范文件：

| 文件 | 性质 | 适用阶段 |
|------|------|---------|
| `开发规范/UI设计与开发规范.md` | 原型/视觉规范 | 设计/原型阶段（HTML + CDN + Tailwind） |
| `docs/engineering/TECH-CONSTRAINTS.md § 1.2` | 工程化规范 | AI 实现阶段（Vue 3 + TypeScript + Element Plus） |

两份规范在技术栈层面存在**冲突**：
- UI 规范：Tailwind 2.x + 原生 JS + CDN + FontAwesome
- TECH 规范：Vue 3 + Element Plus + TypeScript + Pinia

## 2. 决策

🔴 **`docs/engineering/TECH-CONSTRAINTS.md` 是前端规范的唯一权威源（Single Source of Truth）**。

`开发规范/UI设计与开发规范.md` 保留为**视觉设计参考**，但不约束 AI 实现细节。

## 3. 理由

1. **AI 代码生成一致性**：Vue 3 工程栈是 PRD/UX/API 的下游实现基础，AI 必须按此生成代码
2. **避免规范冲突**：两份文件并存会导致 AI 困惑，产生不一致的代码
3. **演进路径清晰**：
   - 原型阶段：使用 `UI设计与开发规范.md` 的视觉规范（色值、字号、布局）
   - 实现阶段：使用 `TECH-CONSTRAINTS.md` 的工程栈（Vue 3 + Element Plus）
4. **降本**：避免维护两套相互冲突的规范

## 4. 影响

### 4.1 对 AI 实现

- AI 严格遵循 `TECH-CONSTRAINTS.md § 1.2`（Vue 3 + TypeScript + Element Plus + Pinia）
- AI 可**参考** `UI设计与开发规范.md` 的视觉规范（如 `#1890ff` 主色），但需在 Element Plus 主题中体现
- AI **不得**使用 Tailwind、原生 JS、CDN 等 UI 规范中的技术栈

### 4.2 对 UX 文档

- UX 文档（`docs/product/UX-*.md`）应**显式标注**视觉规范引用：
  ```
  视觉规范：参考 [开发规范/UI设计与开发规范.md § 三、色彩系统]
  实现栈：[TECH-CONSTRAINTS.md § 1.2]
  ```
- UX 文档中的 CSS class 需映射到 Element Plus 组件

### 4.3 对 `UI设计与开发规范.md`

- 文件保留在 `开发规范/` 目录
- 不被 `docs/` 下的任何 AI 引用
- 仅在设计/原型阶段使用
- 一期不修订、不入 git main 分支的强制审查

## 5. 视觉规范映射（Vue 3 视角）

为避免视觉规范"丢失"，下表将 `UI设计与开发规范.md` 的关键视觉规范映射到 Element Plus 主题变量：

| UI 规范 | 色值/尺寸 | Element Plus 变量 |
|--------|----------|-------------------|
| 主色 | `#1890ff` | `--el-color-primary` |
| 成功色 | `#67c23a` | `--el-color-success` |
| 警告色 | `#e6a23c` | `--el-color-warning` |
| 危险色 | `#f56c6c` | `--el-color-danger` |
| 信息色 | `#909399` | `--el-color-info` |
| 页面背景 | `#f0f2f5` | `--el-bg-color-page` |
| 卡片背景 | `#ffffff` | `--el-bg-color` |
| 边框色 | `#ebeef5` | `--el-border-color-lighter` |
| 主要文字 | `#303133` | `--el-text-color-primary` |
| 次要文字 | `#606266` | `--el-text-color-regular` |
| 辅助文字 | `#909399` | `--el-text-color-secondary` |

> 实施：在 `styles/element-variables.scss` 中覆盖以上变量。
