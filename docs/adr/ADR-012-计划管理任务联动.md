---

## 1. 背景

完整 PRD §5.9 计划管理仅有目录项，无独立 FR/API；`plan/index.vue` 曾为 Mock。用户要求实现 SOP + IP 组 + 赛事 + 步骤分配 + 状态流转。

> **编号说明**：原草稿文件名 `ADR-011-计划管理任务联动.md` 与 `ADR-011-手机管理去除实名人字段.md`（M4）冲突，重编号为 ADR-012。

---

## 2. 决策

1. 新增 `oa_content_plan` / `oa_content_plan_competition` / `oa_content_plan_step` 三表；`oa_task` 增 `plan_id`、`visible_in_list`、`scheduled_start` / `scheduled_end`。
2. 计划草稿期任务状态用 `PLAN_DRAFT`（扩展 STATE-M2 原枚举），并通过 `visible_in_list=0` 双重隐藏。
3. 终止审批：提交后 `TERMINATE_PENDING`；审批人=租户内 `position=OPS_LEADER`（与 seed `oa-leader` 对齐），无独立审批模块。
4. 外部赛事 Phase 1 前端 Mock（`src/mock/competition.ts`），不入库外部 ID 校验。
5. 每步骤多执行人 → 每 `(node, assignee)` 生成一条 `oa_task`。

---

## 3. 后果

- 任务列表 API 默认 `visible_in_list=1`。
- 在 `dict_plan_status` 注册计划状态（DRAFT / IN_PROGRESS / TERMINATE_PENDING / TERMINATED）。
- 任务扩展状态 `PLAN_DRAFT` / `TERMINATED` 已写入 STATE-M2 §1 附录。
