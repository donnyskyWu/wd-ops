# ADR-024：M4 手机与手机卡字段扩展（V85）

**状态**：已采纳  
**日期**：2026-06-15  
**迁移**：`V85__m4_phone_sim_enhancements.sql`  
**关联**：FR-M4-003 · FR-M4-004 · PRD-M4 §4.3/§4.4

## 背景

运营需在手机资产中记录设备影像、采购批次、手机类型、奥创标识；手机卡需补充「损坏」「丢失」状态以反映实物不可用场景。

## 决策

### 1. 手机表 `oa_phone` 新增字段（V85）

| 字段 | 类型 | 控件 | 说明 |
|------|------|------|------|
| `settings_screenshot_key` | VARCHAR(512) | `<ImageUpload />` | 设置页截图；存储 key，API 返回 `settingsScreenshotUrl` |
| `front_image_key` | VARCHAR(512) | `<ImageUpload />` | 正面照 |
| `back_image_key` | VARCHAR(512) | `<ImageUpload />` | 背面照 |
| `purchase_batch` | VARCHAR(64) | `<Input />` | 购买批次 |
| `purchase_date` | DATE | `<DatePicker />` | 购买日期 |
| `purchase_time` | TIME | `<TimePicker />` | 购买时间 |
| `handler_name` | VARCHAR(64) | `<Input />` | 经手人（文本，非 UserSelect） |
| `device_number` | VARCHAR(64) | `<Input />` | 设备编号 |
| `is_aochuang` | VARCHAR(8) | `<DictSelect dict-type="dict_yes_no" />` | 是否奥创手机 |
| `phone_type` | VARCHAR(32) | `<DictSelect dict-type="dict_phone_type" />` | Android / iPhone |

**不变**：`keeper_id` 仍用 `<UserSelect />`（ADR-011）；手机表**不**绑 `realname_id`。

### 2. 手机卡状态字典扩展

`dict_sim_status` 新增：

| label | value |
|-------|-------|
| 损坏 | `DAMAGED` |
| 丢失 | `LOST` |

列表/表单使用 `<DictSelect dict-type="dict_sim_status" />`；`phone_id` 强关联不变。

### 3. 图片上传

复用 `POST /admin-api/oa/file/upload`（`bizType=content` 或通用上传）；手机字段仅存 `*_key`，详情/列表由后端拼 `*_url`。

## 限制

- 经手人为自由文本，不做用户强关联（与保管人职责分离）
- 图片仅支持已接入的上传管线格式与 5MB 上限（与富文本图片一致）

## 影响

| 层 | 变更 |
|----|------|
| DB | V85 迁移 |
| API | `PhoneCreateReq` / `PhoneUpdateReq` / `PhoneRespVO` 扩展；`GET .../phone/list` 支持 `phoneType` 筛选 |
| UI | `PhoneManage.vue` 列表列 + 表单分区（基础信息 / 采购信息 / 影像） |
| Spec | PRD-M4 · UX-M4 · API-M4 · GLOBAL-CONVENTIONS `dict_phone_type` |
