# ADR-011：手机管理去除实名人关联，保管人改 UserSelect

| 字段 | 值 |
|------|---|
| 编号 | ADR-011 |
| 标题 | `oa_phone` 表单不再维护 `realname_id`；保管人使用 `<UserSelect />` |
| 状态 | ✅ Accepted |
| 日期 | 2026-06-11 |
| 决策人 | 产品（实现偏差对齐） |
| 关联 PRD | `PRD-M4-账号管理.md` · FR-M4-003 |
| 关联 Slice | SLICES-M4 S-04 |

---

## 1. 背景

原 PRD FR-M4-003 手机管理数据项含 `realname_id`（`<RealNameSelect />`）及列表列「实名人」。实现中 `PhoneManage.vue` 已移除实名人选择与展示；保管人由 `el-input-number` 硬编码 seed 用户改为 `<UserSelect />`。

实名人与手机的业务关联仍通过**平台账号**（`oa_internal_account.phone_id` + `realname_id`）维护，手机资产表本身不重复绑定实名人。

---

## 2. 决策

### 2.1 手机 CRUD 表单

| 变更 | 说明 |
|------|------|
| **移除** `realnameId` | 创建/编辑/API 均不传实名人 |
| **保留** `phoneNumber` | 新增时 `<Input />` 直输 11 位；编辑时脱敏只读 |
| **保管人** | `keeperId` 必填，`<UserSelect />` 选择 `sys_user` |

### 2.2 列表列

- 展示：手机号（脱敏）、手机编码、机型、保管人、绑定微信、状态
- **不展示**实名人列

### 2.3 API

- `PhoneCreateReq` / `PhoneUpdateReq`：无 `realnameId` 字段
- `keeper`（或 `keeperId`）`@NotNull`，须为本租户有效用户

---

## 3. 影响

| 层 | 变更 |
|----|------|
| 前端 | `PhoneManage.vue` |
| PRD/UX/API | FR-M4-003、P-M4-005、§3 手机 API |
| 测试 | 移除实名人相关 AC；补充 UserSelect 保管人 AC |

---

## 4. 备选方案（未采纳）

| 方案 | 原因 |
|------|------|
| 手机表保留 `realname_id` | 与平台账号强关联重复，运营维护成本高 |
| 保管人手动输入 userId | 违反强关联选择器铁律 |
