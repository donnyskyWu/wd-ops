# ADR-048: M10 企微采集通道

> **状态**: 已采纳 | **决策日期**: 2026-06-22
> **决策人**: 产品 + 开发团队
> **关联**: [ADR-014](./ADR-014-M8-配置管理数据模型.md) · [ADR-045](./ADR-045-M10-奥创多账号与设备同步.md) · [ADR-047](./ADR-047-M4-平台账号凭证SSOT与Collector映射.md) · [M10-三通道采集计划](../delivery/M10-三通道采集计划.md)

## 背景

M10 Phase 2 三通道架构中，**Channel-C（企微）** 采用 `WeComAdapter` + 凭证 SSOT `oa_wework_account`，**不经** `oa_collector_account_bind`（[ADR-047](./ADR-047-M4-平台账号凭证SSOT与Collector映射.md) §3）。

M1 微信分析 Tab（v9.1 §5.40.2）要求企微日聚合：`total_friends`、`today_friend_interactions`、`today_messages_sent`，落库表 `oa_wework_daily_stats`。

本 ADR 记录 2026-06-22 产品决策 Q1~Q7，并作为 **M10-WECOM-S-01** 实现 SSOT。

## 产品决策（Q1~Q7）

| # | 问题 | **决策** | 说明 |
|---|------|----------|------|
| **Q1** | MVP 采集数据范围 | **A — 日聚合 only** | 对齐 M1 微信分析 Tab + v9.1 §5.40.2；不含外部联系人明细 / 客户群 / 会话存档 |
| **Q2** | 凭证 SSOT | **A — `oa_wework_account`** | M8 企微 Tab + M4 企微 Tab 同源；`secret_encrypted` AES-256；`access_token` **应用级内存缓存**（`WeComApiClient`，不落库） |
| **Q3** | 任务 `account_id` 语义 | **B — FK → `oa_wework_account.id`** | 一任务 = 一 corp/agent 应用；采集时展开该应用下 **全部** `oa_wework_employee`（非 per-employee 任务行） |

| **Q6** | 执行路由 | 见 §路由枚举 | `source=WECOM_API` · `platform_type=WEWORK` · `method=INTERNAL` · `data_type=WECOM_DAILY_STATS` |
| **Q7** | 切片顺序 | S-01→S-04 | 本文 §切片 定义 AC；**S-02~S-04 不在 S-01 会话实现** |

## 企微 OpenAPI 映射（MVP）

> 官方文档摘录见仓库外 uploads（92113-0.md / 92132-1.md / 90200-2.md）。`gettoken` 为调用前置（corp_id + secret），非三文档正文但为企微标准凭证接口。

| 文档 path | API | 方法 | OA 用途 | M1 / 落库字段 |
|-----------|-----|------|---------|---------------|
| **（标准）** | `/cgi-bin/gettoken` | GET | `corpid` + `corpsecret` → `access_token` | 凭证前置 |
| **92113** | `/cgi-bin/externalcontact/list` | GET | 按员工 `userid` 拉取 `external_userid[]` | **`total_friends`** = 各员工列表长度之和（当日快照） |
| **92132** | `/cgi-bin/externalcontact/get_user_behavior_data` | POST | 传入应用下全部 `userid[]` + 当日 `[start_time,end_time]` | **`today_friend_interactions`** ← `chat_cnt`；**`today_messages_sent`** ← `message_cnt` |
| **90200** | `/cgi-bin/user/simplelist` | GET | 按 `department_id` 枚举成员 `userid` | **S-01 不用**（员工 SSOT = `oa_wework_employee`）；S-02+ 可选探活 / 员工补齐 |

### 字段映射说明（M1 对齐）

| M1 字段（§5.40.2） | 企微 API 字段 | 聚合规则 |
|-------------------|--------------|----------|
| `total_friends` | 92113 `external_userid` 列表 | 对任务应用下每个 `oa_wework_employee.wework_user_id` 调用后 **求和** |
| `today_friend_interactions` | 92132 `behavior_data[].chat_cnt` | 多 userid 一次请求返回 **总体** 统计；取当日行 `chat_cnt` |
| `today_messages_sent` | 92132 `behavior_data[].message_cnt` | 同上，取 `message_cnt` |

**已知语义偏差（阻塞级以下）**：M1「好友互动」与 API `chat_cnt`（有主动发消息的单聊总数）非逐字等价；MVP 采用 API  documented 字段映射，S-04 M1 联调时产品可再校准文案。

92132 另返回 `new_apply_cnt`、`new_contact_cnt`、`reply_percentage` 等 — **S-01 不落库**（M1 列表未要求）。

## `CollectExecutionService` 路由（Q6）

| `source` | `platform_type` | `method` | `data_type` | 执行体 |
|----------|-----------------|----------|-------------|--------|
| `WECOM_API` | `WEWORK` | `INTERNAL` | `WECOM_DAILY_STATS` | `WeComAdapter.executeDailyStatsCollect(accountId)` |

**`account_id` 解析**：

```
oa_collect_task (source=WECOM_API, account_id = oa_wework_account.id)
        │
        ├─ load oa_wework_account → corp_id / secret
        ├─ load oa_wework_employee[] (wework_account_id, status=ENABLED)
        │
        ▼
WeComApiClient: gettoken → 92113 (per employee) → 92132 (all userids, today)
        │
        ▼
upsert oa_wework_daily_stats (app-level, stat_date=today)
```

默认 `data_type`：当 `task.data_type` 为空且 `source=WECOM_API` 时默认 `WECOM_DAILY_STATS`。

## 铁律

- 全表 `tenant_id` 隔离
- `secret_encrypted` AES-256 + 响应脱敏（M4 已有）
- 枚举 `@InDict`；跨租户 1504
- 任务关联 `oa_wework_account.id` 须 **选择器 + 后端存在性校验**（1501）— **S-03** ✅ `WeworkAccountSelect`
- **禁止** MVP 走 `oa_collector_account_bind`
