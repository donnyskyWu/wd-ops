---

## 1. 背景

用户需求 2–6 要求：

1. SOP 节点增加业务属性「节点类型」：内容生成 / 内容发布 / 普通节点
2. 计划步骤（任务）可分配赛事
3. 任务执行页与内容记录双向关联；内容生成节点完成任务前须有关联内容且状态为「已完成」
4. 内容编辑页扩展文档类型、赛事联动、IP 组作者、按提示词配置 AI 生成

`GLOBAL-CONVENTIONS` 原 `dict_sop_node_type` 为 7 值（启动/审核/创作/拍摄/剪辑/发布/归档），与用户需求不一致；`oa_content` 与 `oa_task` 无正式关联字段。

---

## 2. 决策

### 2.1 节点类型字典（`dict_sop_node_type`）

**替换**原 7 值为以下 3 值（`@InDict` 校验）：

| label | value | 含义 |
|-------|-------|------|
| 内容生成 | `CONTENT_GENERATION` | 执行页可进入内容创作；完成任务须有关联内容且 `status=COMPLETED` |
| 内容发布 | `CONTENT_PUBLISH` | 执行页走发布类操作（细节待产品补充，本期仅占位 FR） |
| 普通节点 | `NORMAL` | 常规执行，无内容联动 |

- 画布拖拽模板（start/process/audit）**仅影响 UI 默认**（如 audit → `need_review=1`），持久化以 `node_type` 字典为准。
- 属性面板新增 `<DictSelect dict-type="dict_sop_node_type" />`，必填。

### 2.2 任务-赛事分配

- 计划级 `oa_content_plan_competition` **保留**（计划关联赛事池）。
- 步骤表 `oa_content_plan_step` **新增** `competition_id`（VARCHAR，外部赛事 ID，来自计划赛事池；**强关联**选择器，非手输）。
- 生成 `oa_task` 时 **继承** 对应 step 的 `competition_id`；任务执行页、内容创作页只读展示赛事信息。

### 2.3 任务-内容关联

- `oa_content` **新增** `task_id`（FK → `oa_task`，可空；任务场景必填）。
- `oa_content` **新增** `competition_id`（外部赛事 ID，可空；由任务继承）。
- `oa_content` **新增** `document_type`（`dict_document_type`，仅 `content_type=ARTICLE` 时必填，见 §2.4）。
- 内容生成节点：执行页首次进入内容创作时 `POST /content/create` 带 `taskId` + `competitionId`，**1 任务 : 0..1 内容记录**（已有则 `GET` 展示，不重复创建）。
- 任务完成校验：`node_type=CONTENT_GENERATION` → 须存在 `oa_content.task_id=当前任务` 且 `status=COMPLETED`。

### 2.4 文档类型字典（`dict_document_type`，新增）

| label | value |
|-------|-------|
| 短视频文案 | `SHORT_VIDEO_SCRIPT` |
| 新号引流 | `NEW_ACCOUNT_TRAFFIC` |
| 赛后复盘 | `POST_MATCH_REVIEW` |
| 正式方案 | `OFFICIAL_PLAN` |
| 预热前瞻 | `PREHEAT_PREVIEW` |

- 用户需求「文档」映射为现有 `dict_content_type` 值 **`ARTICLE`**（文章）；不新增 `DOCUMENT` 枚举值。
- `content_type=SHORT_VIDEO` 时：自动引用同 `competition_id` 且 `document_type=SHORT_VIDEO_SCRIPT` 的 `ARTICLE` 内容正文（只读引用区）。

### 2.5 内容状态「已完成」（`dict_content_status` 扩展）

新增 value **`COMPLETED`（已完成）**：

- 任务执行场景：`保存` → `DRAFT`；`确认` → `COMPLETED`。
- **不触发**三级审核与自动发布（与 `PENDING_FIRST_REVIEW` 路径独立）。
- 任务完成门禁使用 `COMPLETED`；后续是否从 `COMPLETED` 进入三级审核 → **阻塞项 BLK-M2-006**。

### 2.6 AI 生成与提示词

- 文档生成、短视频生成均依赖 M8 `FR-M8-007` 提示词配置；按 **`competition_id` + `document_type`（或 content_type）** 选取模板并做变量替换。
- 具体 AI HTTP 契约（端点、SSE、模型路由）**不在本 ADR 定义**；占位 FR **FR-M2-003-AI**（见 PRD §8 阻塞清单）。
- 视频最终文件：有上传则用上传 URL；否则用 AI 生成视频 URL 作为 `final_video_url`（字段名待 Flyway，阻塞 BLK-M2-007）。

### 2.7 外部赛事 API（BLK-M2-004 已决，2026-06-12）

- 外部源：`http://110.42.49.224:48088/app-api/match`（list + filter/competitions/flat）。
- 平台契约：`GET /admin-api/oa/match/list`、`GET /admin-api/oa/match/leagues`（后端转发，规避 CORS）。
- 计划/任务/内容中的 `competition_id` = 外部 **`scheduleId`**（VARCHAR）；展示名存 `competition_name` 快照。
- 外部 API **无 tenant**；代理与选择器不做租户隔离。
