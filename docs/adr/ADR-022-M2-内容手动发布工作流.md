# ADR-022：M2 内容手动发布工作流

**状态**：已采纳（2026-06-14，用户显式需求）  
**替代**：ADR-017 §BR-023「终审通过 → @Async 自动发布」

## 背景

原 Spec 假定审核通过后系统自动调用平台 API 发布（30s 内变为 `PUBLISHED`）。用户要求改为：

1. 二级审核（或最后一级审核）通过 → `PENDING_PUBLISH`（待发布）
2. 运营人员点击「发布」，选择平台 + 多账号
3. 逐账号调用平台发布适配器
4. 全部成功 → `PUBLISHED`

## 决策

| 项 | 决策 |
|----|------|
| 末级审核通过后状态 | `PENDING_PUBLISH`，**不**自动发布 |
| 无审核配置时提交 | 直接进入 `PENDING_PUBLISH`（非 `PUBLISHED`） |
| 发布 API | `POST /content/{id}/publish` |
| 发布选项 | `GET /content/{id}/publish-options`（含 `publish_enabled=1` 的账号） |
| 账号发布权限 | `oa_account.publish_enabled`；dev 环境有 Cookie 的账号默认启用 |
| 平台 API | `PlatformPublishAdapter` 接口 + Phase 2 `DevStubPlatformPublishAdapter` mock 成功 |
| 审计 | `oa_content_publish_record` 记录每账号发布结果 |
| 权限 | `oa:content:publish` |

## 状态机变更

```
PENDING_SECOND_REVIEW --approve--> PENDING_PUBLISH --publish--> PUBLISHED
（一级关闭时 L2 同理；两级均关闭时 submit-review --> PENDING_PUBLISH）
```

## 后果

- 需更新 `STATE-M2`、`API-M2` 中自动发布描述
- 已发布内容保持只读（与现有一致）
- M10 真实平台 API 接入时替换 `DevStubPlatformPublishAdapter` 实现
