# ADR-049: M10 全量 dataType 任务与采集数据展示桥接

> **状态**: 已采纳 | **日期**: 2026-06-24
> **决策人**: 产品 + 开发团队
> **关联**: [ADR-047](./ADR-047-M4-平台账号凭证SSOT与Collector映射.md) · [ADR-048](./ADR-048-M10-企微采集通道草案.md) · [PRD-M10](../product/PRD-M10-数据采集.md) · [PRD-M1](../product/PRD-M1-运营管理.md)

## 背景

Channel-A 多平台采集 MVP 落地后，各平台需采集多种 `dict_collect_data_type`（粉丝、作品/笔记列表与明细等）。运营若按类型分别建任务，配置成本高且易漏采。

M1 账号分析 / 内部内容分析 / M4 平台账号详情原先主要读 `oa_account_status_log`、`oa_content_daily` 等手工或补录表；M10 采集表（V116/V121/V122）写入后，分析页需 **只读桥接** 展示，避免重复 ETL。

本 ADR 记录 2026-06-24 实现行为，作为 Spec 增量 SSOT。

---

## 产品决策

| # | 问题 | **决策** | 实现 |
|---|------|----------|------|
| **Q1** | 单任务是否覆盖平台全部 dataType？ | **A — 是** | `oa_collect_task.data_type` 为空或 `ALL` 时，`CollectPlatformDefaults.resolveExecutionDataTypes` 按平台顺序 **串行执行全部类型** |
| **Q2** | 任务编辑页是否暴露 method/source/dataType？ | **B — 否（运营侧）** | `task-edit.vue` 隐藏三字段；后端 `CollectTaskServiceImpl` 按 `platformType` 自动填充 `method=INTERNAL` + 默认 `source`；前端只读展示「采集范围」标签 |
| **Q3** | 多类型执行日志如何表达？ | **A — PARTIAL + typeResults** | 全部成功 → `SUCCESS`；全部失败 → `FAILED`；部分成功 → `PARTIAL`；`oa_collect_log.result_json` 含 `typeResults[]`（V120） |
| **Q4** | M1/M4 如何消费采集表？ | **A — CollectedDataQueryService 桥接** | 读 M10 表合并进既有 API 响应；`dataSource=COLLECT` 标识来源；**不**复制写入 `oa_content_daily` |
| **Q5** | 粉丝日聚合 vs 粉丝明细优先级？ | **A — 日聚合优先** | `listFollowerStats` 先查 `oa_account_status_log`；无日聚合时公众号/抖音回退粉丝明细表 |

## 平台默认 dataType 顺序（`CollectPlatformDefaults`）

| `platform_type` | 默认 `source` | 全量 `dataTypes`（顺序执行） |
|-----------------|---------------|------------------------------|
| `WECHAT_OFFICIAL` | `WECHAT_MP_API` | `MP_FOLLOWER_STATS` → `MP_FOLLOWER_LIST` → `MP_ARTICLE_LIST` → `MP_ARTICLE_STATS` → `MP_ARTICLE_CONTENT` |
| `WECHAT_VIDEO` | `WECHAT_CHANNELS_API` | `FOLLOWER_STATS`* → `WECHAT_VIDEO_LIST` → `WECHAT_VIDEO_STATS` |
| `DOUYIN` | `DOUYIN_OPEN_API` | `FOLLOWER_STATS`* → `DOUYIN_FOLLOWER_LIST` → `DOUYIN_VIDEO_LIST` → `DOUYIN_VIDEO_STATS` |
| `KUAISHOU` | `KUAISHOU_OPEN_API` | `FOLLOWER_STATS`* → `KUAISHOU_VIDEO_LIST` → `KUAISHOU_VIDEO_STATS` |
| `XIAOHONGSHU` | `XIAOHONGSHU_OPEN_API` | `FOLLOWER_STATS`* → `XIAOHONGSHU_NOTE_LIST` → `XIAOHONGSHU_NOTE_STATS` |
| `BILIBILI` | `BILIBILI_OPEN_API` | `FOLLOWER_STATS`* only |
| `WEWORK` | `WECOM_API` | `WECOM_DAILY_STATS`（ADR-048） |
| `WECHAT_PERSONAL` | `AOCHUANG_API` | **空**（奥创 Channel-B 未在本切片接任务执行） |

> \*`FOLLOWER_STATS` 为执行层哨兵：路由至 collector 默认 follower-stats（`dataType` 存库为 `null`）。

指定单一 `dataType` 时仅执行该类型（API/IT 保留能力；运营 UI 不暴露）。

## 展示桥接（`CollectedDataQueryService`）

**支持平台**（Channel-A）：`WECHAT_OFFICIAL`、`WECHAT_VIDEO`、`DOUYIN`、`KUAISHOU`、`XIAOHONGSHU`。

| 消费方 | 方法 / 场景 |
|--------|-------------|
| M1 账号分析 | `AccountAnalysisServiceImpl` — 粉丝数、作品数 |
| M1 粉丝分析 | `listFollowerStats` |
| M1 作品 / 内部内容分析 | `pageCollectedContents` / `pageInternalContents` / `aggregateStats` |
| M4 平台账号详情 | `PlatformAccountServiceImpl` — 内容趋势、作品计数 |

**合并规则**：

- 采集平台且 M10 表有数据 → 优先展示采集快照，`dataSource=COLLECT`
- 否则沿用原 `oa_content_daily` / 补录 / API 逻辑（ADR-M1-001）
- 企微 / 个微 / Bilibili 内容列表 **不在** 本桥接范围（企微走 ADR-048 专用 API；Bilibili MVP 仅粉丝统计）

## Out of Scope（本 ADR 不覆盖）

- FR-M10-002 数据质量检查模块
- 奥创 Channel-B 任务执行与 `WECHAT_PERSONAL` 全量类型
- AppSecret OpenAPI 混合采集（ADR-047 §5 Phase 2）
- 全 Channel-A 平台生产 collector 人工联调签收
- 采集数据 → `oa_content_daily` 自动同步（需单独 ADR）
