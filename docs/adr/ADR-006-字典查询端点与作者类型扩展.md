---

## 1. 背景

1. **现状**：后端 `sys_dict_type / sys_dict_data` 已建表（V1 baseline），多个迁移（V2/V3/V5/V6/V7/V8/V10/V12/V14/V17/V19/V21/V24）共写入 **20+** 字典项，但**没有暴露给前端的查询端点**。
2. **PRD 要求**：M0–M9 多处用 `<DictSelect dict-type="dict_xxx" />` 渲染下拉/单选/标签（M1 至少 4 处、M8 4 处、M0 首页大屏口径/M5 财务科目等）。
3. **当前症状（2026-06-09 用户反馈）**：作者管理页「新增作者」保存失败 `authorType: 字典值不合法`——前端硬编码 `LIVE/SHORT_VIDEO/BOTH`，后端 `@InDict("dict_author_type")` 仅放行 `SHORT_VIDEO/ARTICLE`。
4. **M1-V18 seed-ops 现有数据**：`SHORT_VIDEO / ARTICLE` 两类作者。

---

## 2. 决策

### 2.1 新建 `DictController`：GET `/admin-api/oa/dict/data`

| 项 | 做法 |
|----|------|
| 路径 | `GET /admin-api/oa/dict/data` |
| 入参 | `type` (String, 必填, 字典 type) |
| 出参 | `{ list: [{ dictType, label, value, sort, status }], total }` |
| 权限 | `@PreAuthorize` 任意已认证用户可读（不下放到未登录） |
| 租户 | 不区分租户（字典为公共主数据） |
| 缓存 | 命中本地 5 分钟 Caffeine 缓存（key=`type`），减少 DB 命中 |
| 排序 | 按 `sort ASC, value ASC` |

### 2.2 `dict_author_type` 扩展 `LIVE` / `BOTH` / `IMAGE_TEXT`（V-extend 路径）

- V17 已有：`SHORT_VIDEO`（sort=1）、`ARTICLE`（sort=2）
- 新增：`LIVE`（sort=0）、`BOTH`（sort=3）、`IMAGE_TEXT`（sort=4）
- 命名对齐：保留 `ARTICLE` 不改名（已被 V18 seed-ops 引用 9103/9105），同时新增 `IMAGE_TEXT` 为新代码首选；前端渲染按 label，不影响业务。
- 落地方式：**新写一份 V27 Flyway 迁移**，**不修改 V17**（V17 已被生产数据库执行过，违反铁律）。

### 2.3 前端通用 `DictSelect` 组件

- 路径：`ops-platform-ui-vue/src/components/DictSelect.vue`
- 行为：`mounted` 时拉 `/oa/dict/data?type=xxx`，缓存到 `useDictStore`（Pinia），同 type 二次读取 0 网络。
- 失效：登录态变更 / 切换租户时清空。

### 2.4 范围（D3-all）

一次性暴露**全部**字典查询，覆盖以下模块的 `<DictSelect>` 用法：

| 字典 type | 模块 | 典型场景 |

## 3. 取舍

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| A1. 新建 DictController + 写 spec | 横切所有模块，一次到位；字典治理规范 | 1 个新端点 + 1 个新组件 | ✅ 采用 |
| B. 前端写静态常量 | 不动后端 | 字典变化要发版；违反「Spec 驱动」 | ❌ |
| C. 等 PRD 确认 | 严谨 | 阻塞所有页面开发 | ❌ |

`LIVE` / `BOTH` 是否进字典？

| 方案 | 含义 |
|------|------|
| V-keep | 只保留 SHORT_VIDEO / ARTICLE，前端放弃 LIVE/BOTH |
| **V-extend** ✅ | 补 LIVE / BOTH（V27 追加），覆盖直播类作者 |
| V-defer | 暂不决定 |

选 **V-extend**：PRD 字典 select 本就是动态渲染，前端无 `LIVE / BOTH` 体验与运营直播业务不符。

## 4. 影响面

- **后端**：
  - 新增 `DictController` + `DictDataRespVO` + 缓存
  - 新增 V27 Flyway 迁移（仅追加 `dict_author_type` 三条 + 一个 `dict_anchor_type` LIVE 补全）
  - 不动 V17（铁律二）
  - 新增 IT：`DictControllerIT`（覆盖 M1/M2/M8/M9 至少 5 个字典 type）
- **前端**：
  - 新增 `DictSelect` 组件
  - `Author.vue` 等所有 `<el-radio-group>` / `<el-select>` 涉及字典的下拉改造为 `DictSelect`
  - 临时保留 `Author.vue:formatNumber` 的 `v-if` 判空保护（不依赖本次后端改动）
- **GATE 影响**：
  - 不阻塞 S0–S7 已通过 Gate
  - 补一个轻量级 Patch Gate：`P-GATE-DICT-20260609`（跑 `DictControllerIT` 即可），归档 `docs/delivery/gates/P-GATE-DICT-报告-20260609.md`

---

## 5. 兼容性 & 风险

| 风险 | 缓解 |
|------|------|
| 字典值改名/重命名影响历史数据 | 仅追加，不删改；V18 中 `ARTICLE` 保留 |
| 高频调用 | Caffeine 5 分钟缓存；空 type 仍走 SQL 但不预热 |
| 多租户误用 | 字典本身不做 tenant 过滤（公共主数据），但读取走 `DevAuthFilter` 验证登录 |
| Dev mode token 失效 | `request.ts` 已注入 VITE_API_TOKEN，不影响 |

---

## 6. 反向不兼容

- **无**。V27 是追加迁移，不动任何已有表结构、不删任何已有字典值。
- 唯一行为变化：作者类型字典多 3 项可选值；前端 `Author.vue` 改为从接口取，原硬编码移除。
