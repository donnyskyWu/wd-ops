# ADR-026：M9 钉钉工作通知与业务事件去重（V88）

**状态**：已采纳  
**日期**：2026-06-15  
**迁移**：`V88__m9_notification_event.sql`  
**关联**：FR-M9-007 · ADR-013

## 背景

业务通知需触达执行人/审核人/组长：站内消息 + 钉钉。定时监控扫描（粉丝/作品阈值）会重复命中同一业务对象，须去重避免骚扰。

## 决策

### 1. 双通道推送顺序

```
站内消息 (sys_message)
    ↓
钉钉工作通知 asyncsend_v2（主通道，点对点）
    ↓ 失败或未配置
钉钉群机器人 Webhook（可选降级）
```

**工作通知前置**（`oa.dingtalk`）：

| 配置项 | 说明 |
|--------|------|
| `enabled` | `true` |
| `client-id` / `client-secret` | 企业内部应用 |
| `corp-id` | 企业 ID |
| `agent-id` | 工作通知应用 AgentId（示例 `4335523092`） |

接收人使用 `sys_user.ding_user_id`；缺失则跳过工作通知并记 warn 日志。

**机器人降级**（`oa.dingtalk.robot`，可选）：

- `enabled` + `webhook-url` + `secret`
- 仅在工作通知失败或跳过时尝试

跳转链接：`oa.notification.platform-base-url` + 业务路径。

### 2. 业务事件类型（`NotificationEventType`）

| event_type | 触发场景 | 接收人 | category |
|------------|----------|--------|----------|
| `TASK_PENDING` | 内容计划启动，任务变 PENDING | 任务执行人 | BUSINESS |
| `CONTENT_REVIEW_SUBMIT` | 内容提交一级/二级审核 | 配置的审核人 | BUSINESS |
| `CONTENT_REVIEW_APPROVED` | 全部审核通过 | 内容创建人 | BUSINESS |
| `WORK_HIT` | 外部作品播放≥100万；或内部作品 `is_hit=1` | IP 组组长 | ALERT |
| `WORK_LOW_SCORE` | 外部作品完播率 &lt; 20% | IP 组组长 | ALERT |
| `ACCOUNT_HIGH_FANS` | 粉丝数 ≥ 阈值（M8 配置或默认 100万） | IP 组组长 | ALERT |
| `ACCOUNT_LOW_FANS` | 粉丝数 ≤ 阈值（默认 1万） | IP 组组长 | ALERT |

定时扫描：`MonitorAlertScanner`，cron 默认 `0 0/30 * * * ?`（`oa.notification.monitor-scan-cron`）。

### 3. 去重表 `sys_notification_event`（V88）

| 列 | 说明 |
|----|------|
| `tenant_id` + `event_type` + `biz_key` | **唯一键**；先 `tryClaimEvent` 插入成功才发送 |
| `recipient_user_id` | 可选，审计用 |
| `biz_key` 示例 | `task:{id}:PENDING`、`content:{id}:review:{stage}:{reviewerId}`、`account:{id}:ACCOUNT_HIGH_FANS:{statDate}` |

重复扫描/重复审核提交：**静默跳过**（不二次推送）。

### 4. 与 ADR-013 边界

- **本期**：手动钉钉组织同步 + 工作通知推送
- **Phase 2**：钉钉 OAuth 登录、同步删除用户、定时自动同步
