| 状态 | ✅ Accepted |

---

## 1. 背景

Spec 缺口（阻塞项已在本 ADR 闭合）：

| 缺口 | 说明 |
|------|------|
| PRD/UX 无钉钉同步 | FR-M9-001 仅描述用户 CRUD，未定义钉钉集成 |
| API-M9 无部门 API | 列表参数含 `deptId`，但无部门 CRUD / 同步端点 |
| 无 `sys_dept` 表 | `sys_user` 无 `dept_id` / `ding_user_id` |

产品明确要求：用户管理页左侧部门树 + 钉钉部门/人员同步。

---

## 2. 决策

### 2.1 数据模型

- 新增 `sys_dept`：`tenant_id`, `parent_id`, `name`, `ding_dept_id`, `sort`, `status`
- `sys_user` 扩展：`dept_id`, `ding_user_id`
- 钉钉凭证存配置（`oa.dingtalk.*`），**不入库、不提交 git**

### 2.2 API（前缀 `/admin-api/oa/system/dept`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/tree` | 部门树 |
| POST | `/create` | 本地 CRUD |
| PUT | `/update` | 本地 CRUD |
| DELETE | `/delete?id=` | 本地 CRUD（有子部门或用户则 1502） |
| POST | `/sync-dingtalk` | 全量同步钉钉部门树 |
| POST | `/sync-dingtalk-users` | 按已同步部门拉取钉钉用户 |

用户 API 扩展：`GET /user/list?deptId=`；create/update 支持 `deptId`。

### 2.3 同步规则

- **部门**：以 `ding_dept_id + tenant_id` 幂等 upsert；保留本地-only 部门（`ding_dept_id` 为空）
- **用户**：以 `ding_user_id + tenant_id` 幂等 upsert；`username = ding_user_id`；新用户默认角色 `OPS_OPERATOR`；不删除本地已有用户
- **鉴权**：`oa:dept:*` / `oa:user:sync-dingtalk` 权限点；Dev Token 从 DB 加载

### 2.4 钉钉 API

- Token：`POST https://api.dingtalk.com/v1.0/oauth2/accessToken`
- 子部门：`POST /topapi/v2/department/listsub`
- 部门用户 ID：`POST /topapi/user/listid`
- 用户详情：`POST /topapi/v2/user/get`

---

## 3. 不在范围

- 钉钉 OAuth 登录 / SSO（Phase 2，见 ADR-003）
- 同步删除本地用户
- 定时自动同步（仅手动触发）
