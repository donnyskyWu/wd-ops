## 1. 背景

S-14 按 ADR-019 实现后，产品反馈核心偏差：

1. **模板不应存储导入文章的完整正文**，应仅存储 **样式与结构骨架**。
2. **套用模板不应将模板 body 复制到内容**，应将 **内容已有正文** 注入模板版式。

行业惯例（公众号一键排版、电商详情模块、Word 样式）均为 **内容与版式分离**。详见 PROPOSAL v2 §3。

---

## 2. 决策（草案 — 待产品确认）

### 2.1 模板 SSOT：`layout_schema`（非正文文档）

| 层 | 字段 | 格式 | 用途 |
|----|------|------|------|
| 模板表 | `layout_schema` | JSON v2（结构 + 样式 + 槽位） | 模板编辑 SSOT |
| 模板表 | `style_css` | 消毒后 CSS 子集（可并入 schema） | 全局/块默认样式 |
| 模板表 | `preview_html` | 消毒 HTML | 列表/预览 **骨架**（占位符，非用户正文） |
| 模板表 | `schema_version` | INT | `2` = 本 ADR |
| 模板表 | `layout_json` | JSON v1 | **Deprecated**；只读兼容 + 迁移源 |

**内容表**字段不变：`body`（语义正文 SSOT）、`layout_json`（套用 **输出** 实例）、`layout_html`、`body_format`、`layout_template_id`。

### 2.2 内容保真规则（铁律）

> **套用模板时，不得增删改 `body` 中的语义文字。** 仅允许分段映射、样式注入、固定装饰块插入。

- `body` 在套用前后 **字符串一致**（允许 `\n` 分段解析，不允许改字）
- 模板内 `demoText` / 导入样本文字 **不得** 写入内容
- 若 `body` 为空 → 拒绝套用（错误码 **2013**），**待确认** OQ-v2-04

### 2.3 导入语义

| 来源 | 输出 | 禁止 |
|------|------|------|
| URL / docx / paste | `layout_schema` + `extraction_report` | 将样本段落 text 持久化为模板正文 |
| 提取步骤 | StyleExtractor → SlotNormalizer → strip text | 全文 `parseHtmlToLayout` 直接入库 |

导入预览须展示 **骨架**，UX 免责声明见 PROPOSAL §4.5。

### 2.4 套用语义

**v1（废弃）**：`content.layout_json ← template.layout_json`（copy）

**v2（本 ADR）**：

```
content.layout_json ← LayoutMergeService.merge(
    body: content.body,
    existingLayout: content.layout_json?,  // 换模板时先 extractText
    schema: template.layout_schema
)
content.layout_html ← render(content.layout_json)
content.body        ← unchanged
content.layout_template_id ← template.id
content.body_format ← LAYOUT
```

- 提供 **preview** 端点（不写库）供 UX 确认
- `overwrite`：仅表示是否覆盖已有 **layout 实例**，**不**表示覆盖 `body`

### 2.5 `layout_schema` 槽位模型（v2 最小集）

| 块类型 | 说明 |
|--------|------|
| `heading` | 样式 + level；套用时可映射首段或 Markdown 标题 **（OQ-v2-05 待确认）** |
| `slot` + `slotKind=paragraph` + `repeat:true` | 顺序吃掉 body 分段 |
| `slot` + `quote` / `list` | 可选槽；映射策略待确认 |
| `divider` / `frame(image)` | 固定或可选装饰 |
| `styleRef` | 指向 `globalStyles` 键 |

完整 JSON 示例见 PROPOSAL §6.1。

### 2.6 与 ADR-019 保留部分

以下 ADR-019 决策 **继续有效**（本 ADR 不修订）：

- 内容/模板双字段：`layout_json` + `layout_html`（内容侧）
- 类型匹配：`content_type=ARTICLE` + `document_type` 可空规则
- HTML sanitize、租户隔离、权限模型
- 编辑器选型 BLK-M2-015（TipTap 等）
- URL 抓取 BLK-M2-012 Fallback

### 2.7 模板新增/编辑 UX（P-M2-015）

模板编辑页 **必须** 从「完整文档块编辑」改为「样式/槽位编辑」：

- 表单 SSOT 字段：`layout_schema`（非 `layout_json`）
- 编辑器：块类型序列 + `globalStyles` / `styleRef`；段落为 **repeat 槽**，不录入营销正文
- 预览：`preview_html` 骨架示意；默认 **无** 可发布 demo 文案（OQ-v2-07 选项 C）
- 服务端保存校验：拒绝未标记的用户正文写入 schema
- Legacy `schema_version=1` 模板：只读 + 提示迁移，禁止按 v1 继续编辑全文

详见 PROPOSAL v2 §14 Q1、§7。

### 2.8 内容表 `oa_production_content` 数据语义

**不新增表列**（默认）；修订字段语义：

| 字段 | v2 语义 |
|------|---------|
| `body` | 语义正文 SSOT；套用 **不改字** |
| `layout_json` | merge **输出实例**（`version: 2`，含真实 text） |
| `layout_html` | 由实例渲染 |
| `layout_template_id` | 套用来源审计 |
| `body_format` | 套用后 `LAYOUT` |

可选在 `layout_json` 根级嵌入 merge metadata（`mergedAt`、`overflowSegmentCount` 等），**不强制独立列**。

已有 v1 套用记录 **不自动迁移**；提供「按正文重新套用」操作（extractText → merge）。

详见 PROPOSAL v2 §14 Q2、§9.2。

### 2.9 系统预置模板（PRESET）

**决策（草案）**：允许通过 Flyway seed 预置 **版式骨架模板**，加速冷启动。

| 项 | 约定 |
|----|------|
| `source_type` | 新增字典值 `PRESET` |
| 内容物 | 仅 `layout_schema` + `preview_html` 骨架；**无** 营销全文 |
| 状态 | 默认 `ENABLED` |
| 租户 | **OQ-v2-11 待确认**；默认每租户 seed（`tenant_id` 与 `seed-base` 一致） |
| 首批清单 | 8 款（长文导读、活动预告、种草清单、卖点卡片、赛事战报、FAQ、数据复盘、引流贴片）— 产品可裁减 |
| 生命周期 | 预置条目不可删，可停用；租户 **复制后** 编辑副本 |
| 合规 | 对标 135/淘宝 **模块结构**，自研样式，不嵌入第三方 HTML |

规格 SSOT：`docs/delivery/SEED-M2-公推预置模板草案.md`  
实现切片：S-14e 或独立 Seed 任务（**依赖** ADR-020 Accept + `layout_schema` 列）。

## 4. 备选方案（未采纳）

| 方案 | 弃用原因 |
|------|----------|
| 继续 copy，UX 提示用户手动改字 | 与行业惯例相反；AI 生成工作流不可用 |
| 模板仅存 CSS，套用客户端 merge | 审核/查看须服务端一致渲染 |
| 套用时用 AI 重写正文 | 违反内容保真；须人工审核，Out of Scope |
