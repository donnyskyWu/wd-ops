## 反思

1. **聚合放后端是铁律**：B1 教训——前端 list.reduce 累加永远有分页错位风险，所有 KPI 必须在后端 SQL 聚合
2. **导出端点先行**：spec AC 明示导出，必须在 S-R2 阶段就建 `/export`，不能留到 S-R6
3. **维度参数必须端到端**：B2 教训——前端 DictSelect 给了选项但后端 controller 不收，等于死字段，spec 落地必须 4 段对齐（API/前端/IT/seed）
4. **替换时小语法错误代价高**：S-R6 期间替换 handleExport 函数时多粘了一个 `}` 导致整页 vite compile error 必须 reload → 教训：大段替换后必须立刻 reload 验证
5. **增长率双重计算（×100）**：后端 IMPLEMENT 习惯返回小数 → 前端显示 % 时别再 ×100，必须明文约定
