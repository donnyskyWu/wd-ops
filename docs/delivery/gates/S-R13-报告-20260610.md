## 六、反思

### 6.1 关键沉淀
1. **"4 字段对照表"在低 bug 密度页也有效**：本来想"这页没 bug 不需要走查"——S-R12 沉淀的方法论强制走一遍后，**至少发现 1 个 B2 冗余** + 确认了 page/size 风格的 1 个一致性边界。**走查不应跳过"看似 OK"的页**。
2. **H2 seed ID 必须用真实范围**：第一遍写 IT 用 1001 → 4 失败。教训：**写 IT 第一时间就 grep 实际 seed 文件确认 id 范围**。可作为 H2 IT 模板的第 1 条。
3. **同 slice 内不同模块可用不同分页命名**：SOP 用 `pageNum`，M1 AccountAnalysis 用 `page/size`——**后端 controller `@RequestParam` 严格按名匹配是事实正确**。SOP-R11 报告里说"前端要统一为 pageNum"是跨模块一致性建议，但**单模块内保持与 spec 一致**也是正确选项。
