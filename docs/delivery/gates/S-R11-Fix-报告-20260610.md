## 五、反思

### 5.1 关键沉淀

1. **H2 NOT NULL + @InDict 双重限制**：用 `JdbcTemplate.update(...)` 直插 seed 是最稳的 IT 写法——避开 Service 层校验链（@InDict → 业务校验 → DB 约束）。**这个模式值得作为团队 IT 模板**。
2. **静态走查 30 分钟暴露 60% bug**：本 slice 总耗时 ≈ 90 分钟，IT 通过 + 后端修复完，**没启 dev**。比"启 dev → 跑一遍 → 才发现"快 3 倍。
3. **`getSopReviewList` 是死代码**：原 `api/sop.ts` 根本没导出，前端 import 是 TS 编译期就该报错的——**TypeScript 类型约束没起作用的根因是 `import *` 模式 + 弱 any 断言**。
