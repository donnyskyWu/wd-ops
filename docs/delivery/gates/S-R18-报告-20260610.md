## 6. 反思

### 6.1 走查方法论再升级

- **B20/B21 的 2 个 SyntaxError 是"端到端才能发现"的真 bug**——4 字段对照 + API export 检查**抓不到**（API 文件存在，export 名字不一致）
- 建议：5 字段对照**升级为 6 字段** = 加 **`grep "import.*from '@/api/<file>'"` 与 `export function` 一一比对**
- 实操：在 L-α 静态分析时，task/index.vue 的 `import { submitReview }` 看起来 OK（没拼写错误），但 `task.ts` 实际是 `submitTaskReview`——需要"双向 grep"

### 6.2 Meta-test 的价值

T3~T7 是**文件内容静态检查**——能在 0.01s 内验证"L-α 的修复实际生效"。**没有这些 meta-test**，L-α 走查报告中的"已接 API"无法被独立验证。**值得保留**。

### 6.3 "L-α 后 E2E 自然崩"是可预测的

4 轮 23 失败归类：**真实 bug 3 + 测试需更新 5 = 8 个 root cause**。L-α 改动 19 页时**没跑 E2E**（走查完没回归），所以全量崩了。建议**"walkthrough 完成 → E2E 必跑"**作为新规约。
