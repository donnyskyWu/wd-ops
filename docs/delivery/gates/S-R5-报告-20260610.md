## 2. 真因

### 2.1 补录提交 mock（`InternalContent.vue:176-183`）

```javascript
const handleImportSubmit = async () => {
  if (!importFormRef.value) return
  await importFormRef.value.validate(async (valid: boolean) => {
    if (!valid) return
    ElMessage.success('补录数据已提交，等待审核')  // ← 硬编码
    importDialogVisible.value = false
  })
}
```

**S-R2-Fix 报告 Q3 只补了"趋势去 mock"，没补"补录去 mock"**（S-R2-Fix 报告内 TODO 漏列）

### 2.2 审核通过不合并（`InternalContentServiceImpl.java:111-123`）

```java
public void reviewImport(Long id, ImportReviewReq req) {
    ContentDataImportDO existing = requireImport(id);
    // ... 改 reviewStatus
    contentDataImportMapper.updateById(existing);
    // ← 缺：spec AC-M1-006-2 要求合并入 oa_content_daily
}
```

**spec 写了，service 没实现**。M1InternalContentTrendIT 只测了 trend 端点，**没测"审核通过后 trend 能查到该日"**（覆盖盲区）

### 2.3 我的补录入口缺失
`InternalContent.vue` 完整没"我的补录"按钮。后端 API 全部闲置。

### 2.4 搜索栏 5 项筛选项失效
`InternalContentService.list(platformType, dataSource, pageNo, pageSize)` 只收 4 个，**前端多发了 `ipGroupId/keyword/importType/startDate/endDate`**，Spring 忽略 → 筛选项看起来失效。

### 2.5 切 Tab 时 pagination.total 不重置
`handleTabChange` 只重置 pageNo，total 跨 Tab 累加（切到抖音 total 还是 26 因为 26 是全部总数）。

### 2.6 补录类型硬编码 vs dict 真值
- 后端 dict 真值：`API_EXCEPTION` + `OFFLINE`（V17__m1_ops_core.sql L113-114）
- 前端 `IMPORT_TYPE_CONFIG` 写：`API_EXCEPTION` + `ACCOUNT_BANNED` + `OFFLINE_PRACTICE` + `OTHER`（**2 个 dict 没有**）

## 5. 反思 + TODO

### 5.1 IT 覆盖延迟（再次踩 S-R3 反模式 #6）

**这次 S-R5 是 IT-first**（改 service 前先写 IT），但漏掉了 trend-merge IT 早期没写。补救路径：
- ① service 改完 → ② 写 IT `reviewApprovedMergesToContentDaily` → ③ 跑 IT 验证 trend 真有数据 → ✅ Green

**比 S-R4 进步**：S-R4 是"后补" IT，S-R5 是"service 改完立刻写 IT"，只是不完美（理想是**先 Red 写 IT → 再改 service**）。

- spec §AC-M1-006-2 写"合并入 oa_content_daily，data_source='IMPORT'"，但**实际表 schema 无 data_source 字段**

- S-R5 按 spec 走：**合并到 oa_content_daily（readCount 真值），不标 data_source**
