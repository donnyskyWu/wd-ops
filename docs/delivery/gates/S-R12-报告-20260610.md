## 六、反思

### 6.1 关键沉淀
1. **"3 字段对照表"再显威力**：S-R11 沉淀的"前端 import ↔ api export / api URL ↔ controller / controller @RequestParam ↔ 前端 params"——**这次抓到了 controller 端点缺失 + service 缺失 + DTO 缺失** 3 个层级的问题。S-R11 错过 1 个：还要加 **"前端用到的 type 字段 ↔ 后端 VO/DO 字段"** 第 4 列对照。
2. **`/list` vs `/{id}` 路由冲突**：Spring MVC 路径匹配**精确优先**——`@GetMapping("/list")` 必须排在 `@GetMapping("/{id}")` 之前才不会被 "list" 字符串当 id。这是 S-R3 教训（S-R3 修过 FollowerAnalysis）的同型 bug。
3. **S-R11 教训升级到 v2**：报告自查 7 条之外，**还应当 verify "IT 失败信息 vs 实际数据真值"**——`NumberFormatException: "list"` 直接告诉你是路径被字符串匹配。

### 6.2 留给下个 slice 的 TODO
- **浏览器复测**：dev 起来后 agent-browser 验证 4 期望实证
- **B8 (P2 leaderId ↔ leaderUserId)**：待 SOP 走查时统一收口

### 6.3 方法论升级
- **"3 字段对照表" → "4 字段对照表"**：
  1. 前端 import ↔ api export
  2. api URL ↔ controller
  3. controller @RequestParam ↔ 前端 params
  4. **前端 type 字段 ↔ 后端 VO/DO 字段**（S-R12 新增）
