# S-R2-Phase2 报告（2026-06-09）

> **目标**：补 S-R2 走查发现的 1 个 seed 漏字典（`dict_time_dimension`）
> **范围**：1 个 Flyway 迁移 + 后端重启
> **状态**：✅ 后端已重启，字典可用

---

## 1. 根因

S-R2 自验（`M1AnalysisS06IT` 10/10 ✅）**没覆盖字典**。走查时 `FansAnalysis.vue` / `Efficiency.vue` 用 `dict_time_dimension`，后端 `DictController` 严格校验 type 必须存在（`OaErrorCodes.DICT_TYPE_NOT_FOUND=2020`），DB 没 seed → 报"字典 type 不存在"。

**Spec 明确要求**（`docs/engineering/API-M1-运营管理.md` §4.2、PRD-M1-007-2）：
> `timeDimension: DAY/WEEK/MONTH`

**Seed 漏了**——V8~V28 没有任何迁移插入此字典。

---

## 2. 改动文件（1 个）

### 新增
1. `yudao-server/yudao-module-oa/src/main/resources/db/migration/V29__m1_dict_time_dimension.sql`

```sql
-- V29: 补 time_dimension 字典（spec: API-M1-运营管理 §4.2 timeDimension=DAY/WEEK/MONTH）
INSERT INTO sys_dict_type (type, name, status) VALUES
('dict_time_dimension', '时间维度', 'ENABLED');

INSERT INTO sys_dict_data (dict_type, label, dict_value, sort, status) VALUES
('dict_time_dimension', '按日', 'DAY', 1, 'ENABLED'),
('dict_time_dimension', '按周', 'WEEK', 2, 'ENABLED'),
('dict_time_dimension', '按月', 'MONTH', 3, 'ENABLED');
```
