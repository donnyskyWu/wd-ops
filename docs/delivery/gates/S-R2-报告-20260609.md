## 6. 反思（学到的）

1. **API 路径拼写错是 S-R1 漏点**——前端 `/oa/follower/*` 是 S-R1 旧 mock 路径，spec §4.2 写的是 `follower-analysis`。**教训**：前端 API 文件的 URL 必须在 Spec 落地时**逐一对齐** spec.md 的 path，不能凭直觉。
2. **VO 字段命名不一致**（works viewCount vs readCount）是后端 V0 没和前端定契约。**教训**：每个新接口必须有 VO spec + 前端 type 生成器。
3. **后端用 `defaultValue=1` 兜底未声明 param** 会让"错的请求"返回"看似正确的数据"，掩盖 bug。**教训**：后端 controller 应开启 `strict=true` 或删 defaultValue 让未知 param 直接 400。
4. **PowerShell `Invoke-WebRequest` 默认按 GBK 解码**——是测试工具问题但容易误判后端"乱码"。
5. **环境约束自我认知**（本会话反思）：本会话在 headless 终端，不应擅自判断用户本地进程/端口；`netstat` 看到 3080 不应直接改 vite.config.ts。应优先**问用户**而不是改共享配置。已在本次回滚。
6. **S-R2 自验堵住 3 个潜在运行时 bug**：编译期 3 个错（缺 import / 错码名 / 类型转换 / lambda effectively-final）。这些如未自验，到浏览器走查阶段才暴露会更费时。**教训**：每个 slice 自带 mvn test 自验是必要的最小验证。
