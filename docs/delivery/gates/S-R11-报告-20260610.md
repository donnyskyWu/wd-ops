## 三、修复方案

### 3.1 P0 三连（最小修复集合）

**修复 B2/B3（review 端点）**：
- 后端 `SopReviewController` 加 `GET /list` 端点（带 `templateId / reviewStatus / pageNum / pageSize`）
- 或：前端移除 `loadReviewList` 的分页逻辑，改为直接调 `/pending`（按列表渲染）
- 选 B（更轻、且 pending 本身不分页有合理性）

**修复 B2/B5（缺失 import/export）**：
- `api/sop.ts` 补 `getSopReviewList`（= `pending` 的分页包装）或移除 import
- `api/sop.ts` 补 `updateSopTemplateStatus`

**修复 B1/B6/B7（pageNo vs pageNum）**：
- **选项 A（按 spec）**：前端改用 `pageNum`
- **选项 B（兼容性）**：后端 `SopTemplateController.list` 收 `pageNo` 也认（按 spec 改回）
- **决策**：按 `API-M2-内容生产.md` §1.1 spec 显式定义为 `pageNum`——**前端改**为 `pageNum`

### 3.2 P1 配套修复

- **B9**：移除 `.catch(() => Promise.resolve())` 吞错模式，让错误冒到外层 try/catch
- **B8/B11**：浏览器复测后确认字段名，修 types

### 3.3 P2 长期改进

- B10：加 `if (v && v !== 'ALL')` 过滤
- B12：改用 `enum-alias.ts` + DictSelect
- B13：补 4 个核心 IT（M2SopTemplateListIT / M2SopReviewPendingIT / M2TaskListIT / M2TaskMyTasksIT）
