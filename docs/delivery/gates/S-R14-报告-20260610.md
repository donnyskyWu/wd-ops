## 三、修复方案

### 3.1 后端（B2/B3）
- ✅ `KnowledgeController.list` 收 `tags`、`isPublic` 参数
- ✅ `KnowledgeBaseService.list` 方法签名 + impl 加 tags LIKE + isPublic eq 过滤
  - tags 用 `LIKE '%tag%'` 兼容逗号分隔
  - 用 `apply(sql, params)` 动态 SQL 注入

### 3.2 前端（B1/B4/B7）
- ✅ `types/knowledge.ts:72` `pageNo: number` → `pageNum: number`
- ✅ `types/knowledge.ts:82` `isPublic?: boolean` → `isPublic?: 0 | 1`
- ✅ `KnowledgeVO.isPublic: boolean`（前端） / `tags: string[]`（前端）已对（adaptVO 适配）
- ✅ `views/production/knowledge/index.vue:288-293` searchForm 用 `pageNum`
- ✅ loadList 做字段映射：`keyword→title`、`isPublic boolean→0/1`
- ✅ Pagination 绑定 `pageNum`

### 3.3 未修（B8）
- `el-switch` 不能 unset 回到"未选"——Switch 必须有 true/false 双值。**P2 backlog**，建议改用 DictSelect（dict_yes_no）。

## 六、反思

### 6.1 关键沉淀
1. **S-R13 沉淀的"4 字段对照表"威力再显**：S-R14 一遍走查暴露 4 个 P0，**全在字段名错位**。这类 bug 静态分析 < 5 分钟就能全找。
2. **H2 IT 模板升级**：
   - 第 1 条：写 IT 前先 grep seed + 用 unique title/tag（避免 IT 间数据冲突）
   - 第 2 条：H2 IT 不共享事务，每个 IT 必须独立 setup
3. **adaptVO 模式的两面性**：前端已有 `parseTags` / `Boolean(raw.isPublic)` 适配，**但搜索/提交时未做反向适配**——半成品适配是 S-R1 时为赶进度留的债。
4. **`el-switch` UX 局限**：用 v-model + handleReset 设 undefined，**不能 unset**——必须 DictSelect 才支持 3 态。
