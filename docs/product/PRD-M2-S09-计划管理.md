## FR-M2-009 计划管理

### 描述

业务计划关联 SOP 模板、IP 组与外部赛事，保存时自动生成计划任务（草稿期隐藏），启动后任务进入任务列表。**每 SOP 步骤须分配 ≥1 个赛事**；**每步骤 1 名执行人**（2026-06-13）。

### 主流程

1. 选择 SOP 模板、IP 组、多个赛事（外部 API `/admin-api/oa/match/list`，**BLK-M2-004 已决**）
2. 自动加载 SOP 节点，逐步分配**赛事（多选）**、**执行人（单选）**及起止时间（默认=计划日期）
3. 保存 → 计划状态=草稿，任务状态=PLAN_DRAFT 且 `visible_in_list=0`
4. **编辑草稿** → 可修改计划名称、日期、赛事池、步骤赛事与执行人（仅 `DRAFT`）
5. 启动计划 → 计划=进行中，任务=PENDING 且可见
6. **多赛事步骤** → 按「赛事 × 执行人」生成多条 `oa_task`，各带 `competition_id`
7. **计划详情** → 展示计划信息 + **生成的任务记录表**（节点、赛事、执行人、岗位、状态、scheduled_start/end）
8. 申请终止 → 计划=终止审批中；运营组长审批 → 计划/任务=已终止

### 验收标准

**AC-M2-009-1** 保存草稿后任务列表不可见该计划任务  
**AC-M2-009-2** 启动后任务列表可见且状态=PENDING  
**AC-M2-009-3** 终止须组长审批通过后计划与任务均为 TERMINATED  
**AC-M2-009-4** 每步骤分配赛事后，关联任务 `competition_id` 属于步骤赛事集合  
**AC-M2-009-5** 草稿计划可编辑名称/日期/赛事池/步骤，非草稿编辑拒绝  
**AC-M2-009-6**（2026-06-13）步骤选 2 赛事 + 1 执行人 → 生成 2 条任务  
**AC-M2-009-7**（2026-06-13）计划详情抽屉展示任务列表及计划起止时间列
