## 1. 概述

### 1.1 一句话描述

运营数据分析平台的**全局仪表盘**，以 IP 组为筛选核心，聚合 4 大核心指标 + 2 个图表 + 待办提醒 + 快捷入口。

### 1.2 目标与指标

| 维度 | 目标 | 可量化指标 |
|------|------|------------|
| 效率 | 替代打开多个报表的耗时 | 1 秒内看到全平台核心指标 |
| 决策 | 快速发现异常 | 待办/告警延迟 ≤ 1h |
| 统一入口 | 90% 的常用功能从首页进入 | 快捷入口点击率 ≥ 60% |

### 1.3 术语表

| 术语 | 定义 |
|------|------|
| **IP 组筛选** | 全局筛选维度，作用于所有指标区域；通过 `<IpGroupTreeSelect />` 选择 |
| **数据权限过滤** | 按角色 + 部门 + IP 组 + 人员四级，详见 `BR-006` |
| **快捷入口** | 基于用户权限动态渲染的菜单卡片 |
| **待办提醒** | 跨模块聚合的提醒（SOP 待审 / 发布待审 / 绩效待算 / 集成异常） |

### 2.1 角色 × 能力

| 能力 \ 角色 | 系统管理员 | 运营管理者 | 运营组长 | 运营人员 | 数据分析师 | 数据/财务 |

| 快捷入口 | ✅ 全部 | ✅ 全部 | ✅ 本组 | ✅ 本组 | ✅ 数据相关 | ✅ 财务相关 |

### 2.2 权限规则

- **数据范围**：BR-006（角色+部门+IP组+人员 4 级）
- IP 组筛选器自动只显示用户有权限的 IP 组（不显示则看不到）
- 快捷入口动态渲染（无权限则入口不可见）

## 4. 功能需求

### FR-M0-001 首页仪表盘

#### 4.1.1 描述

打开平台第一眼看到的"驾驶舱"，基于 IP 组筛选实时聚合关键数据。

#### 4.1.2 前置条件

- 用户已登录
- 用户拥有"首页-查看"权限

#### 4.1.3 主流程

1. 用户登录后默认进入首页
2. 顶部操作栏加载 IP 组选择器（`<IpGroupTreeSelect />`），默认"全部"
3. 加载 4 个指标卡（总作者数 / 内容总数 / SOP 完成率 / 平均绩效等级）
4. 加载 2 个图表（内容发布趋势折线图、平台分布饼图）
5. 加载待办提醒表格（最多 10 条）
6. 加载快捷入口（基于权限动态显示）

#### 4.1.4 备选/异常流

| 异常 | 提示 |
|------|------|
| 接口失败 | Banner "首页加载失败" + 重试按钮 |
| 用户无任何 IP 组权限 | IP 组筛选器仅显示"全部" |
| 无待办 | 空插图 "暂无待办" |
| 无快捷入口 | 不显示该卡片 |

#### 4.1.5 业务规则

- **IP 组筛选联动**：选择 IP 组后，所有指标 + 图表 + 待办统一刷新
- **一次性加载**：首次进入加载所有数据，**不轮询**
- **手动刷新**：右上角"刷新"按钮触发全量重新加载
- **数据来源**：API 采集 + IMPORT 补录（聚合）

#### 4.1.6 数据项

#### 4.1.6.1 顶部操作栏

| 控件 ID | 类型 | 文案/默认值 | 关联字典/实体 |
|---------|------|------------|---------------|
| F-IP | `<IpGroupTreeSelect />` | "IP 组筛选"（默认"全部"） | `oa_ip_group`（仅启用状态 + 用户权限范围内） |
| F-DATE-RANGE | `<DateRangePicker />` | "开始日期 - 结束日期"（默认近 7 天） | - |
| BTN-REFRESH | 按钮 | "刷新" | - |

#### 4.1.6.2 统计卡片

| 字段 | 类型 | 数据来源 | 计算公式 |
|------|------|----------|----------|
| total_authors | Integer | `oa_author` | COUNT(DISTINCT id) WHERE status=1 AND tenant_id=? |
| total_content | Integer | `oa_content` | COUNT(*) WHERE publish_time BETWEEN ? AND ? |
| sop_completion_rate | Decimal(5,2) | `oa_task` | SUM(CASE WHEN status='DONE' THEN 1 ELSE 0 END) * 100.0 / COUNT(*) |
| avg_perf_grade | String | `oa_perf_record` | AVG(grade) → S/A/B/C/D |

#### 4.1.6.3 图表

**内容发布趋势（折线图）**：
- X 轴：日期
- Y 轴：内容数量
- 维度：可选平台（`<DictSelect dict-type="dict_platform_type" />`）

**平台分布（饼图）**：
- 分类：`dict_platform_type`
- 指标：内容数量
- 显示：占比 %

#### 4.1.6.4 待办提醒

| 字段 | 类型 | 来源 |
|------|------|------|
| title | String | 各模块 |
| source | String | SOP / 发布 / 绩效 / 集成（`<DictSelect dict-type="dict_alert_type" />`） |
| time | DateTime | 触发时间 |
| action_url | String | 跳转 URL |

#### 4.1.6.5 快捷入口

| 字段 | 类型 |
|------|------|
| name | String |
| icon | String |
| url | String |
| permission | String（`oa:xxx:view`） |

#### 4.1.7 验收标准（AC）

**AC-M0-001-1**（默认进入）
- Given 用户已登录
- When 打开首页
- Then 4 个指标卡、2 个图表、待办、快捷入口全部加载完成 ≤ 1.5s

**AC-M0-001-2**（IP 组筛选联动）
- Given 进入首页，默认 IP 组=全部
- When 在 IP 组选择器中选择"娱乐八卦大组/八卦一组"
- Then 4 个指标卡 + 2 个图表 + 待办列表均按"八卦一组"重新聚合

**AC-M0-001-3**（IP 组选择器仅显示启用 + 有权限）
- Given 用户是"运营组长"、归属 IP 组"八卦一组"
- When 打开 IP 组选择器
- Then 仅显示用户有权限的、状态=启用的 IP 组；"娱乐八卦大组/八卦二组"（无权限）不显示

**AC-M0-001-4**（字典选择）
- Given 内容发布趋势图
- When 切换"平台"筛选为"抖音"
- Then 折线图仅展示抖音平台的内容数

**AC-M0-001-5**（手动刷新）
- Given 首页已加载
- When 点击"刷新"
- Then 重新调用全部 5 个接口，loading 完成后展示新数据

**AC-M0-001-6**（数据权限）
- Given 用户角色=数据分析师
- When 进入首页
- Then 快捷入口中"账号管理""IP 组管理"等不显示；仅显示"数据报表""指标管理"等
